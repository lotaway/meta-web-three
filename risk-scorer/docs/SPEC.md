1. 项目目标

主要目标

构建可解释的信用风险评分模型，用于预测用户违约概率（PD）

将违约概率转化为标准化信用分数

输出评分卡规则、业务可读的“拒绝/审批/复审”策略

生成模型输出的解释指标（特征贡献/分项得分）

业务子目标

支持新客户（申请/BNPL）与已贷客户风险评估模型

输出可监控的性能指标（AUC、KS、PSI）

确保特征与风险结果具有业务意义与解释逻辑

2. 非目标（必须排除）

直接用大语言模型（LLM）做最终评分结果（可用于解释辅助，但不能替代评分模型）

使用完全黑盒模型作为单一风险决策依据（如单纯深度神经网络无解释能力）

3. 输入/输出规范

输入

结构化特征表（用户画像、行为标签、信贷历史等）

必须包含清晰可信定义的目标标签（违约/未违约）

输出

模型文件（训练好模型 + 分箱信息 + 特征编码方案）

评分卡映射表（变量区间 → 分数）

指标报告（AUC、KS、IV、PSI）

决策输出（审批/复审/拒绝）

4. 模型要求与约束

可解释性约束

模型必须令特征系数/分数可解释，并能业务复述

用 WOE/IV 等方法处理特征离散化与解释能力提升

性能验收标准

AUC ≥ 0.7（可根据业务调整）

KS ≥ 0.3（若适用于评分卡评价指标）

数据分布稳定性

PSI 监控，保证训练与线上评分特征分布一致

逻辑约束

分箱后各分箱的 WOE 值应呈单调性或能解释风险变化趋势

各变量 IV 值必须大于设定阈值（排除无区别变量）

5. 验收方式
项目	验收指标	通过条件
模型预测效果	AUC、KS	分别通过预设阈值
分箱质量	WOE 单调性、IV	有较合理分布无异常单元
可解释性	分数卡输出 & 说明报告	每个变量得分业务可解释
决策策略输出	审批/复审/拒绝	满足定制信用阈值策略
线上表现监控	PSI、坏账比率	不超过阈值波动
6. 验收数据样本行为

目标标签定义

按业务定义的违约标准生成 target（二分类例：逾期> X天视为违约）

观察窗口设定需明确（如 6 个月/一年）

7. 验收样例输出格式
```yaml
model:
  type: logistic
  version: vYYYYMMDD
  metrics:
    auc: 0.75
    ks: 0.35
features:
  - age
  - income
  - woe_bin_of_utilization
score_card:
  age:
    - bin: [18,25]
      value_woe: -0.25
      score: 12
```
