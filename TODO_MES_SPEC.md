一、产品定位与核心原则
1.1 产品定位
配置化主导型MES是一套“可配置的业务中台”，而非“写死的应用软件”。它以元数据驱动、配置优先为核心理念，让实施人员和客户通过后台界面完成80%的系统适配工作。

1.2 核心设计原则
原则	说明
元数据驱动	数据模型、界面布局、业务规则均通过元数据定义
配置优先	任何客户需求，优先考虑“能否配置实现”，而非“能否开发实现”
流程可编排	提供可视化流程设计器，支持业务流程的拖拽式配置
数据可扩展	预置扩展字段机制，客户可自定义字段
规则可定义	业务规则（如防错、校验、触发）通过规则引擎配置

3.2 数据配置能力
3.2.1 实体扩展字段机制
每个核心业务实体（工单、产品、物料、设备等）必须支持动态扩展字段：

yaml
扩展字段定义:
  字段类型:
    - 文本（单行/多行）
    - 数字（整数/小数）
    - 日期/时间
    - 下拉单选/多选
    - 复选框/开关
    - 关联引用（关联其他实体）
  
  配置项:
    - 字段编码（唯一标识）
    - 字段名称
    - 默认值
    - 必填/非必填
    - 唯一性约束
    - 校验规则（正则/范围）
    - 是否列表显示
    - 是否可搜索
  
  适用实体:
    - 生产工单（mes_pro_work_order）
    - 产品定义（mes_pro_product）
    - 物料清单（mes_pro_bom）
    - 设备台账（mes_eq_device）
    - 质检单（mes_qc_ipqc）
3.2.2 数据字典/下拉选项配置
系统提供统一的数据字典管理，所有下拉选项均可配置：

选项值、标签、排序

选项分组

选项依赖关系（如“选择A后，B选项自动过滤”）

选项禁用/启用

3.2.3 编码规则配置
支持单据编码的完全可配置：

yaml
编码规则配置:
  规则要素:
    - 固定前缀（如"MO"、"QC"）
    - 日期占位符（YYYYMMDD / YYMM / 等）
    - 流水号（起始值、步长、填充长度）
    - 业务字段引用（如产品类型、工厂代码）
    - 分隔符
  
  示例: "MO-[工厂代码]-[YYYYMMDD]-[流水号4位]"
  
  配置范围:
    - 工单编码
    - 质检单编码
    - 物料批次编码
    - 设备编码
    - 产品SN规则
3.3 流程配置能力（核心差异化能力）
这是配置化MES最关键的能力。

3.3.1 可视化流程设计器
提供B/S架构的拖拽式流程设计器：

功能	要求
节点类型库	基础节点、设备交互、数据处理、系统集成、人工任务、子流程
连接方式	串行、并行、条件分支、循环
节点配置	节点属性、输入/输出参数映射、超时设置、异常处理策略
版本管理	流程模板的版本控制，支持回滚
引用机制	流程模板可被工序/工位引用，每次引用创建独立实例
3.3.2 典型流程场景配置
系统预置以下流程模板，支持快速配置：

流程类型	说明	可配置点
生产报工流程	工位完成报工	报工数据字段、质检触发、物料消耗扣减、产成品入库
异常呼叫流程（Andon）	异常上报与逐级处理	呼叫类型、上报层级、超时升级规则、处理人分配
来料检验流程	IQC检验	抽样方案、检验项、合格/不合格判定规则、处置流程
设备点检流程	设备日常点检	点检项、点检周期、异常上报
返修流程	不良品返修	返修路由、重检流程、工时统计
3.3.3 节点属性配置示例
yaml
流程节点示例（PLC交互节点）:
  节点类型: "设备交互节点"
  属性配置:
    - 交互类型: "读/写/等待信号/发送指令"
    - 设备地址: "变量引用/常量" [citation:9]
    - 超时时间: "5000ms"
    - 超时策略: "重试(3次)/跳过/终止流程"
    - 异常处理: "触发Andon/记录日志/发送通知"
  
  变量映射:
    输入变量: "流程变量 -> 设备地址"
    输出变量: "设备返回值 -> 流程变量"
3.4 规则配置能力（业务规则引擎）
3.4.1 防错规则引擎
支持可视化的条件-动作规则配置：

yaml
防错规则示例:
  场景: "SMT上料防错"
  
  触发条件:
    - 事件: "物料条码扫描"
    - 条件表达式: "扫描物料PN != BOM指定物料PN"
  
  触发动作:
    - 行为: 
        - "禁止通过"
        - "显示错误提示: '物料不匹配，请检查'"
        - "记录异常日志"
        - "可选: 触发Andon呼叫"
  
  配置方式: 后台界面条件选择器 + 表达式编辑器
3.4.2 质检规则配置
配置项	说明	示例
抽样方案	支持GB/T 2828.1-2003标准	一次正常、加严、放宽
AQL配置	可接受质量水平	主要缺陷0.65，次要缺陷1.5
判定规则	合格/不合格判定逻辑	缺陷数≤Ac判合格，≥Re判不合格
处置规则	检验结果触发动作	合格→入库，不合格→隔离待评审

6.1 区分“配置”与“定制开发”的边界
分类	是否算“配置化”	实现方式

修改质检标准上下限	✅ 是	后台配置
